# Examples

Native iceoryx2 applications packaged as ROS 2 (`ament_cargo`) packages so
they build with colcon and link against the Rust message crates generated by
`rosidl_generator_rs`. Each subdirectory is one example; see its own README
for what it demonstrates and how to run it. This document covers the build
setup common to all of them.

All commands assume a sourced ROS 2 environment, e.g. inside the development
distrobox (see [../README.md](../README.md)). In the following, `<repo>` is
the iceoryx2 checkout, `<workspace>` is your colcon workspace, `<example>` is
the example's package directory under `examples/`, and `<distro>` is your
ROS 2 distribution (e.g. `jazzy` or `humble`).

## Prerequisites

* The cargo-aware colcon build types and installer:

  ```bash
  pip install colcon-cargo colcon-ros-cargo
  cargo install cargo-ament-build
  ```

* A ROS 2 install space providing the generated Rust message crates **with
  the `serde` feature**. The crates shipped with a distribution's debs may be
  generated by an older `rosidl_generator_rs` without it, so build fresh
  ones in your workspace for the message packages your example uses:

  ```bash
  cd <workspace>/src
  git clone https://github.com/ros2-rust/rosidl_rust.git
  # clone the interface packages providing your messages, e.g.:
  #   git clone -b <distro> https://github.com/ros2/common_interfaces.git
  cd <workspace> && source /opt/ros/<distro>/setup.bash
  colcon build --packages-up-to <message packages> rosidl_generator_rs
  ```

* Two symlinks tying the example and the workspace together:

  ```bash
  # 1. Make the package discoverable by colcon.
  ln -s <repo>/integrations/ros2/examples/<example> <workspace>/src/

  # 2. Symlink the ROS install space so it can be discovered by the
  #    Cargo build.
  ln -sfn <workspace>/install <repo>/integrations/ros2/examples/<example>/.ros_install
  ```

## Building

```bash
cd <workspace>
source /opt/ros/<distro>/setup.bash && source install/setup.bash
colcon build --packages-select <example>
```

## Troubleshooting

```console
error: failed to load source for dependency `std_msgs`
  Unable to update <...>/<example>/.ros_install/std_msgs/share/std_msgs/rust
  failed to read `<...>/.ros_install/<...>/Cargo.toml`: No such file or directory
```

The `.ros_install` symlink is missing or points at an install space that
does not provide the generated message crates — (re)create it as described
in [Prerequisites](#prerequisites).
