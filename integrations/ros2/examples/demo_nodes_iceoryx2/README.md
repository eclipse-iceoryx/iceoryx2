# demo_nodes_iceoryx2

Native iceoryx2 demo nodes following the tunnel's bridge contract:
`std_msgs` — using the Rust crate generated by `rosidl_generator_rs`
— serialized as CDR into the payload of the service mapped to the
ROS 2 topic `/chatter`.

See [../README.md](../README.md) for the build setup common to all examples;
this README only covers what is specific to this one.

## Messages

This example uses `std_msgs/msg/String`. Build its generated crate (and
`builtin_interfaces`, a transitive dependency) into your workspace:

```bash
cd <workspace>/src
git clone https://github.com/ros2-rust/rosidl_rust.git
git clone -b <distro> https://github.com/ros2/common_interfaces.git
git clone -b <distro> https://github.com/ros2/rcl_interfaces.git
cd <workspace> && source /opt/ros/<distro>/setup.bash
colcon build --packages-up-to std_msgs rosidl_generator_rs
```

## Building

```bash
colcon build --packages-select demo_nodes_iceoryx2
```

## Running

Outbound (iceoryx2 → ROS 2):

```bash
source <workspace>/install/setup.bash
ros2 run demo_nodes_iceoryx2 talker
# in other shells:
#   cargo run --example tunnel_reactive # or tunnel_polled
#   ros2 run demo_nodes_cpp listener
```

Inbound (ROS 2 → iceoryx2):

```bash
source <workspace>/install/setup.bash
ros2 run demo_nodes_iceoryx2 listener
# in other shells:
#   cargo run --example tunnel_reactive # or tunnel_polled
#   ros2 run demo_nodes_cpp talker
```
